### Types of Load Balancers

| Type | Layer | Use Case | Your Choice |
|------|-------|----------|-------------|
| **ALB** (Application) | Layer 7 (HTTP) | Web apps, APIs, microservices | âœ… This one |
| **NLB** (Network) | Layer 4 (TCP/UDP) | Gaming, IoT, extreme performance | |
| **CLB** (Classic) | Both | Legacy (don't use for new projects) | |
| **GWLB** (Gateway) | Layer 3 | Firewalls, IDS/IPS | |

**Why ALB?**
- Understands HTTP (can route by path, host, headers)
- Built-in health checks
- Perfect for web applications

## 3. alb.tf - The Code

### Complete File

```hcl
########################################
# ALB Security Group (public tier)
########################################

resource "aws_security_group" "alb_sg" {
  name        = "${var.project_name}-${var.environment}-alb-sg"
  description = "ALB HTTP access from internet"
  vpc_id      = module.vpc.vpc_id

  # Inbound: HTTP from the internet
  ingress {
    description = "HTTP from internet"
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = [var.allowed_http_cidr]
  }

  # Outbound: ALB â†’ targets in VPC
  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }

  tags = {
    Project     = var.project_name
    Environment = var.environment
    ManagedBy   = "terraform"
    Name        = "${var.project_name}-${var.environment}-alb-sg"
  }
}

########################################
# Application Load Balancer (public)
########################################

resource "aws_lb" "app_alb" {
  name               = "${lower(var.project_name)}-${var.environment}-alb"
  internal           = false
  load_balancer_type = "application"
  security_groups    = [aws_security_group.alb_sg.id]
  subnets            = module.vpc.public_subnets

  tags = {
    Project     = var.project_name
    Environment = var.environment
    ManagedBy   = "terraform"
    Name        = "${var.project_name}-${var.environment}-alb"
  }
}

########################################
# Target Group for App EC2
########################################

resource "aws_lb_target_group" "app_tg" {
  name        = substr("${lower(var.project_name)}-${var.environment}-tg", 0, 32)
  port        = 80
  protocol    = "HTTP"
  vpc_id      = module.vpc.vpc_id
  target_type = "instance"

  health_check {
    path                = "/"
    protocol            = "HTTP"
    matcher             = "200-399"
    interval            = 30
    timeout             = 5
    unhealthy_threshold = 2
    healthy_threshold   = 2
  }

  tags = {
    Project     = var.project_name
    Environment = var.environment
    ManagedBy   = "terraform"
    Name        = "${var.project_name}-${var.environment}-app-tg"
  }
}

########################################
# HTTP Listener â†’ forward to target group
########################################

resource "aws_lb_listener" "http" {
  load_balancer_arn = aws_lb.app_alb.arn
  port              = 80
  protocol          = "HTTP"

  default_action {
    type             = "forward"
    target_group_arn = aws_lb_target_group.app_tg.arn
  }
}
```

## 5. The Load Balancer

### The Code

```hcl
resource "aws_lb" "app_alb" {
  name               = "${lower(var.project_name)}-${var.environment}-alb"
  internal           = false
  load_balancer_type = "application"

    Name = "${var.project_name}-${var.environment}-alb"
  }
```

### Line-by-Line Explanation

| Attribute | Value | Explanation |
|-----------|-------|-------------|
| `name` | `devops-interview-lab-dev-alb` | Must be unique, lowercase, 3-32 chars |
| `internal` | `false` | Internet-facing (public) |
| `load_balancer_type` | `application` | Layer 7 ALB (HTTP-aware) |
| `security_groups` | `[alb_sg.id]` | Which SG controls traffic |
| `subnets` | `public_subnets` | Where ALB nodes live |

### Why `lower()` Function?

```hcl
name = "${lower(var.project_name)}-${var.environment}-alb"

# var.project_name = "devOps-interview-lab"
```

```hcl
name = substr("${lower(var.project_name)}-${var.environment}-tg", 0, 32)
```

## 10. Interview Questions

### Q1: What's the difference between ALB and NLB?

> **ALB** (Application Load Balancer) operates at Layer 7 (HTTP/HTTPS), understands HTTP headers/paths, and can route based on content. **NLB** (Network Load Balancer) operates at Layer 4 (TCP/UDP), is faster, handles millions of requests per second, but doesn't understand HTTP.

### Q2: Why does the ALB need to be in public subnets?

> An internet-facing ALB needs public IP addresses to receive traffic from the internet and a route to the Internet Gateway to send responses. Only public subnets provide both.

### Q3: What happens if a target fails health checks?

> After failing the `unhealthy_threshold` (2) consecutive health checks, the target is marked unhealthy and removed from the pool. The ALB stops sending traffic to it but keeps checking. Once it passes `healthy_threshold` (2) checks, it's added back.

### Q4: What is a target group?

> A target group is a collection of targets (servers) that receive traffic from the load balancer. It defines the port, protocol, health check settings, and contains the registered targets (EC2 instances, IPs, or Lambdas).

### Q5: Why use `substr()` for the target group name?

> Target group names have a 32-character maximum. `substr(string, 0, 32)` truncates the name to ensure it never exceeds this limit, preventing Terraform errors.

### Q6: What does the listener do?

> A listener is a process that checks for connection requests on a specific port and protocol. When traffic arrives, it uses rules (or the default action) to determine which target group should receive the traffic.

### Q7: Can an ALB have multiple listeners?

> Yes! You can have listeners on different ports (e.g., 80 for HTTP, 443 for HTTPS). Each listener can have its own rules and actions.

### Q8: How does ALB distribute traffic between targets?

> By default, ALB uses round-robin algorithm - it sends requests to each healthy target in turn. It also supports least outstanding requests algorithm for targets with varying processing times.

### Q9: Why is `internal = false` in your config?

> `internal = false` creates an internet-facing ALB with public IPs, accessible from the internet. `internal = true` would create an internal ALB only accessible within the VPC.
